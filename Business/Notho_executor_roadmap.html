Notho phased upgrade roadmap for a single developer, ordered by realistic effort from quick wins to hard infrastructure.

Phase 1 — Quick Wins
Ship in the next 4 weeks
Week 1–4
Week 1
Streak Freeze mechanic
One Supabase column (freeze_count), one UI toggle. Cap at 2/month. Done in a day. Ship it first — highest visible impact, lowest code surface.
Week 1–2
Lifecycle email sequences (D+7, D+14, D+30)
You already have D+1. Add 3 more triggers in whatever email tool you're using (Resend/Postmark). Write the copy yourself — 30 min per email. Don't build a custom drip system; use the provider's scheduled send or a simple Supabase cron edge function. This is not a week of work, it's 2 days max.
Week 2–3
Spaced repetition review system
Store last_seen and ease_factor per lesson per user in Supabase. SM-2 algorithm is 20 lines of TypeScript. Surface as a "Review Due" queue on the home screen. Do not build a full SRS app — one queue, one card UI, one Supabase query. This is the single highest literacy-outcome move in Phase 1.
Week 3–4
Deeper content (investing, tax, estate planning)
Write 2–3 new lessons per topic. Don't build new infrastructure — slot into existing unit structure. Content creation, not engineering. Outsource the writing if budget allows; edit it yourself. This is where mission alignment lives.
Phase 2 — Medium Effort
Ship in months 2–3
Month 2–3
Month 2, wk 1
R49/month subscription (Stripe)
Stripe Checkout, one price ID, a subscriptions table in Supabase, a middleware check on premium routes. Offer a 7-day free trial. Do not build a custom billing portal yet — Stripe's hosted portal handles upgrades/cancellations. You need revenue before anything else in Phase 2 makes sense to build.
Month 2, wk 2–3
PWA offline mode
Next.js + next-pwa. Cache lesson content at install time. This is a connectivity lifeline for your target market. Add a service worker, precache lesson JSON, handle offline gracefully. One week of focused work. This also makes the app installable on Android home screens — free distribution channel.
Month 2, wk 4
Social sharing with real proof
Generate a shareable image (og:image via Vercel OG or satori) showing the user's streak, XP, and a specific achievement. One share endpoint, one designed card. WhatsApp is the primary distribution channel in SA — optimize for that aspect ratio.
Month 3
SMS notifications (Africa's Talking or Clickateli)
Streak reminders only at first. Africa's Talking has a SA number pool and a simple REST API. One edge function, one cron job. Don't build preference management yet — opt-out link only. Cost is negligible at early user counts.
Month 3
B2B employer wellness / ETF referrals
Do not build this. Go find 2–3 employers or a single ETF provider and close a deal manually first. You need a signed LOI before you write a line of B2B code. The tech is a simple coupon/cohort system — one week of work — but only after the commercial agreement exists.
Phase 3 — Hard
Quarter 2 onward — only after revenue
Q2+
Q2
Bank statement PDF parsing
Use an existing parser (pdf2json or a hosted API like Parseur). Build a Supabase edge function that ingests, categorises transactions, and maps to the existing budget planner. Scope to Capitec statements first — largest SA retail bank, consistent PDF format. Don't attempt multi-bank parsing until Capitec works end-to-end. This alone will take a full month for one developer.
Q2–Q3
Native Android app
Do not start this until PWA is live and you have 500+ active users. The PWA buys you 90% of the benefit. If you still need native after validating PWA, use Capacitor to wrap the existing Next.js app — not a rewrite. A full React Native rewrite as a solo developer is a 6-month distraction.
Q3+
Open Banking (Capitec/FNB/Nedbank APIs)
SA Open Banking regulation is still maturing. Capitec has a developer portal but live access requires a formal partnership application. FNB and Nedbank are further behind. Do not block any other work on this. Start the partnership application process now — it takes months — but build nothing until you have sandbox credentials and a signed agreement.
Three traps that will kill velocity:
1. Building B2B features before closing a commercial deal. 2. Starting native Android before the PWA is live. 3. Attempting multi-bank Open Banking before Capitec PDF parsing is in production. Each of these burns a month with nothing to show users.
