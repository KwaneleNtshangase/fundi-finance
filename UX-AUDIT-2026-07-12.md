# Fundi Finance — UX Audit (Persona Walkthroughs)
**Date:** 2026-07-12 · **Target:** https://fundiapp.co.za (production) · **Method:** In-browser persona walkthroughs (first-time visitor, new user onboarding, engaged learner, budgeter, error-prone user, mobile user) at desktop 1400px and mobile 390px widths, cross-checked against the codebase.

---

## P0 — Fix first (blocks core loops or first impressions)

### 1. Level-gated courses never unlock (confirmed bug)
Advanced Investing (req. Level 5), Advanced Tax Planning (7), Business Finance: Advanced (7), and Estate Planning (10) are locked even for a **Level 13** account.

**Root cause (verified):** `src/app/(app)/learn/page.tsx` renders `<LearnView>` without the `userLevel` prop. `LearnView.tsx:664` defaults `userLevel = 1`, so `isLocked` (line 987) is always true for gated courses. The comment in `pageViews.types.ts` says XP gating "gives XP real utility" — right now that entire payoff loop is dead for every user.

**Fix:** pass the real level into `LearnView` (same place `streak` is passed). One-line fix, big retention payoff.

### 2. /login and /signup are unbranded 404 dead-ends
Both URLs return a black page with only "404 — This page could not be found." No logo, no link home. These are the two most commonly typed/linked auth URLs (and the kind ad campaigns and password managers generate).

**Fix:** redirect `/login` and `/signup` → `/`, and build a branded 404 with a "Back to Fundi" button.

### 3. No landing page for logged-out first-time visitors
`/` server-redirects straight to `/learn`, which for a logged-out visitor is just the logo, "Your financial journey starts here," and an auth form. No value proposition, features, screenshots, pricing, or trust signals. A curious visitor from word-of-mouth or social hits a login wall and has no reason to sign up.

**Fix:** a simple marketing landing page (what it is, 3 screenshots, "free" + POPIA/security reassurance, CTA). This is likely your single biggest conversion lever.

### 4. Course progress bars read as "complete" at 0%
The empty track is a light tint of the fill colour, so at a glance "Insurance & Protection — 0/6 lessons" shows what looks like a full bar. Scanning the Learn page, *everything* looks done, which kills the motivation to start.

**Fix:** neutral grey/low-opacity track, coloured fill only for actual progress.

### 5. 5–7 second splash screen on every full load
Every hard navigation shows the logo splash with no progress indicator before the app renders. For a data-conscious SA audience on mobile networks, this feels broken and invites bounces.

**Fix:** skeleton UI instead of splash; cache shell + last-known stats; investigate what's blocking first render (auth + content fetch appear serialized).

## P1 — High-friction annoyances

6. **Onboarding "Skip" doesn't exist.** Copy says "Skip if you prefer" but there's no skip control, and clicking **Next** with nothing selected (or 18+ unchecked) silently does nothing — no error, no shake, no hint. Users will assume the app is frozen. Add a Skip link + inline validation ("Select a goal and confirm you're 18+").
7. **Search fails on multi-word queries and results hide below the fold.** "tax refund" → *No results*, though "Taxes for Individuals" exists ("tax" works). Results also render *below* the Weekly Challenge and Daily Challenges cards, so matches are off-screen — search feels broken even when it works. Tokenise the query, match lesson titles too, and move results directly under the search box.
8. **Locked courses give zero feedback.** Clicking a locked card does nothing, and the badge ("Reach Level 7, Serious Learner") doesn't show your current level or XP remaining. After fixing #1, show "Level 13 ✓ Unlocked" / "2 400 XP to go."
9. **Leaderboard is full of test accounts.** `e2e_test_bot` (rank #1!), `Tester`, `Zuzu`, rows of 0 XP users. Being told "60 XP to overtake E2E_TEST_BOT" destroys credibility. Filter internal/test accounts from prod; seed with real-feeling defaults if population is thin.
10. **Daily Goal shows "90 / 50 XP" with a green→red gradient.** Exceeding the goal overflows the label, and progress toward a goal ending in *red* reads inverted. Cap the bar at 100% with a "Goal smashed! +40 XP extra" state, and make the gradient end in the success colour.
11. **Escape doesn't close modals** (Add Entry, and likely others). Keyboard users and power users expect Esc-to-dismiss. Add Esc + click-outside handling to the modal wrapper.
12. **Budget month oddities.** "Customised for July 2026 / Reset to default" banner shows on a month with **no entries** (confusing — what was customised?), and there's no forward-month chevron, so users can't plan next month.
13. **"Goals" nav doesn't contain your goal.** The Goals tab (route `/quests`) holds challenges + stokvels, while your actual money goal ("Save for a home") lives on Learn and Profile. Rename the tab ("Challenges" or "Quests") or move the goal card in.
14. **Stokvels section shows bare "Loading…" for ~8–10s.** Use a skeleton card and look at the query — it's much slower than the rest of the page.

## P2 — Polish

15. **Pinch-zoom is disabled** (`user-scalable=no, maximum-scale=1` in the viewport meta). WCAG 1.4.4 issue; hurts older and low-vision users — a core audience for financial education. Remove those attributes.
16. **Quiz options look like plain text.** No visible card/border until selected, so tap targets are ambiguous. Add subtle borders/hover states.
17. **Hearts aren't visible inside a lesson** — the one place you can lose them. Show hearts in the lesson header.
18. **Mobile parity:** streak/hearts/XP ("My Stats") aren't visible anywhere on the mobile Learn page. Consider a compact stats strip.
19. **Learn page is a very long single column** (~14 course cards). Group by tier (Basics / Growing / Advanced) or use a 2-col grid on desktop to aid discovery.
20. **Copy nits:** "1 units" → "1 unit"; goal deadline renders as "By Sept" (no year).
21. **Import dialog:** "Preview transactions" looks tappable before any file is chosen; disable it (with reduced opacity) until a file + consent are present.

## What's already working well (keep these)
Exit-lesson confirmation dialog · inline answer explanations after each quiz question · POPIA consent + "processed in memory only" copy on statement import · clear empty states with single CTAs ("Add your first entry") · mobile bottom nav · consistent "not financial advice" disclaimers · calculator flow with chart + save-to-profile.

## Suggested order of attack
1. Pass `userLevel` to `LearnView` (one line, restores the whole XP loop)
2. Landing page + `/login`/`/signup` redirects + branded 404
3. Progress-bar track colour + Daily Goal cap
4. Splash → skeleton (perf pass)
5. Onboarding skip/validation + search fixes
6. Scrub test accounts from the leaderboard
